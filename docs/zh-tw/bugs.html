問題回報

遇到錯誤或想提出新功能嗎？我們在 GitHub 上追蹤所有問題。

回報錯誤

在 GitHub 上開啟新 issue，說明您遇到的問題細節。

瀏覽現有 Issues

看看您的問題是否已經被回報過，或尋找已知問題的解決方法。

需要提供的資訊

一份完整的錯誤回報能幫助我們更快修正問題。請提供：

裝置型號
例如：Boox Note Air 3、Kobo Elipsa 2E、Kindle Scribe，或一般 Android 手機／平板。
Android 版本
例如：Android 12。可在裝置的「設定 → 關於」中查看。
WebView 版本
可在 EinkBro 的「設定 → 關於」中找到，或於系統設定中的 WebView 項目查看。
EinkBro 版本
例如：15.10.0。顯示於 EinkBro 的「設定 → 關於」。
重現步驟
以編號列出觸發錯誤的確切步驟。越具體越好。
預期行為
您原本預期會發生什麼事。
實際行為
實際上發生的事。如果有任何錯誤訊息，請一併附上。
截圖／螢幕錄影
若是畫面相關的錯誤，截圖會非常有幫助。若適用，也可以使用整頁截圖功能。

常見疑難排解

在回報前，可以先試試這些常見的解決方法：

網頁無法載入或顯示異常

  • 清除快取 — 設定 → 資料控制 → 清除快取
  • 確認 JavaScript 狀態 — 確保 JavaScript 已啟用（設定 → 啟動控制 → JavaScript），或該網站已加入白名單
  • 確認 Cookie 狀態 — 部分網站需要 Cookie。請在「設定 → 啟動控制 → Cookies」中啟用
  • 停用廣告封鎖 — 部分網站會因廣告封鎖而異常。請將該網站加入廣告封鎖白名單
  • 更新 WebView — 過舊的 Android System WebView 可能導致渲染問題。請透過 Google Play 商店或您裝置的應用程式商店更新

效能問題

  • 啟用網頁快取 — 設定 → 行為 → 啟用網頁快取
  • 停用圖片 — 設定 → 啟動控制 → 圖片（在慢速網路下可加快載入）
  • 使用閱讀模式 — 移除沉重的頁面元素以加快渲染
  • 要求節省流量 — 設定 → 啟動控制 → 要求「節省流量」（告訴網站提供較輕量的內容）

翻譯功能失效

  • 確認您已連線到網際網路
  • 若是圖片翻譯，請確認已於「設定 → 其他 → 翻譯圖片 API 金鑰」設定金鑰
  • 若是 AI 翻譯，請在「設定 → GPT」確認您的 API 金鑰

AI 功能失效

  • 在「設定 → GPT」確認您的 OpenAI 或 Gemini API 金鑰
  • 確認模型名稱正確（OpenAI 預設為 gpt-4.1，Gemini 預設為 gemini-2.5-flash）
  • 若使用自架伺服器，請確認自訂主機 URL 可從您的裝置存取

功能請求

有新功能的點子嗎？請在 GitHub 開啟 issue，並在標題前加上 [Feature Request]。請描述：

  • 您希望看到什麼功能
  • 為何這個功能有用（特別是對 E-Ink 裝置使用者而言）
  • 您期望它如何運作

貢獻修正

若您能自行修正錯誤，我們歡迎 pull request：

  1. Fork 此儲存庫
  2. 從 main 建立一個分支
  3. 進行修正並測試（最好在 E-Ink 裝置上實測）
  4. 送出 Pull Request 並引用對應的 issue 編號

請參考開發者指南了解建置說明與架構細節。

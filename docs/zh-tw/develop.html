開發者指南

EinkBro 以 Kotlin 與現代 Android 開發實踐建構。本指南涵蓋架構、建置流程與貢獻者的主要開發領域。

架構概觀

┌─────────────────────────────────────────────────┐ │ app module │ │ │ │ ┌──────────┐ ┌───────────┐ ┌──────────────┐ │ │ │ Activity │ │ ViewModel │ │ Repository │ │ │ │ (Compose) │──│ (StateFlow│──│ (Room, API) │ │ │ └──────────┘ │ LiveData)│ └──────────────┘ │ │ └───────────┘ │ │ │ │ ┌──────────┐ ┌───────────┐ ┌──────────────┐ │ │ │ EBWebView│ │ Koin │ │ Services │ │ │ │ (Custom │ │ (DI) │ │ (Translation │ │ │ │ WebView) │ │ │ │ TTS, AI) │ │ │ └──────────┘ └───────────┘ └──────────────┘ │ ├─────────────────────────────────────────────────┤ │ ad-filter module │ │ JS scriptlets · FilterViewModel · filter logic │ ├─────────────────────────────────────────────────┤ │ adblock-client module │ │ Native C++ via CMake (JNI) │ └─────────────────────────────────────────────────┘

核心模式

  • MVVM — ViewModel 搭配 LiveData/StateFlow 管理 UI 狀態
  • Repository Pattern — 資料層抽象化（例如 OpenAiRepository.kt）
  • Dependency Injection — 以 Koin 為 DI 容器，於 EinkBroApplication.kt 設定
  • Room Database — 本地資料持久化，搭配 KSP 註解處理器
  • Jetpack Compose — UI 元件（對話框、設定、工具列、選單）

主要元件

元件說明

建置指令

開發

# 建置 debug APK
./gradlew assembleDebug

# 建置 release APK
./gradlew assembleRelease

# 建置 releaseDebuggable 變體
./gradlew assembleReleaseDebuggable

# 安裝 debug 版本至已連接裝置
./gradlew installDebug

# 清除建置輸出
./gradlew clean

測試

# 執行單元測試
./gradlew test

# 執行儀表化測試
./gradlew connectedAndroidTest

# 執行 Lint 檢查
./gradlew lint

# 僅 debug 版的 Lint
./gradlew lintDebug

主要開發領域

WebView 客製化

E-Ink 專屬的 WebView 最佳化實作於 EBWebView.kt：

  • 依 E-Ink 重新整理特性調整的自訂滾動行為
  • 注入閱讀模式 CSS，呈現乾淨的內容
  • 為原生功能整合提供 JavaScript 介面（觸控追蹤、頁面資訊等）
  • JS 檔案放在 assets/，於執行期載入

AI 整合

ChatGPT／AI 功能橫跨多個元件：

  • GptActionsActivity.kt — AI 聊天互動的 UI
  • OpenAiRepository.kt — OpenAI、Gemini 與自架伺服器的 API 通訊
  • ChatGptQuery.kt — 儲存查詢歷史的 Room 實體
  • 使用者可透過系統提示定義自訂 GPT 動作

EPUB 處理

使用 EPubLib 實作完整的 EPUB 讀取與寫入：

  • 於瀏覽器內解析並呈現 EPUB 內容
  • 將網頁匯出為包含圖片與多章節的 EPUB
  • 目錄可編輯，支援拖曳排序

翻譯服務

多家提供者的翻譯整合，支援段落級、整頁與圖片翻譯。提供者包括 Google Translate、DeepL，以及透過 OpenAI／Gemini 的 AI 翻譯。

資料庫與 Schema

Room 資料庫 schema 存放於 app/schemas/。修改實體時請務必定義好遷移路徑。主要實體包括書籤、歷史紀錄、GPT 查詢與白名單。

國際化

本專案支援 25 種以上語言，透過 Crowdin 管理。翻譯檔使用標準 Android 格式，位於 res/values-*/strings.xml。請勿直接編輯翻譯檔 — 所有變更都應透過 Crowdin 進行。

ProGuard

Release 版本會透過 ProGuard 進行程式碼混淆與最佳化：

  • app/proguard-rules.txt — Release 版本
  • app/proguard-rules.pro — ReleaseDebuggable 版本

如何貢獻

  1. 於 GitHub 上 Fork 此儲存庫
  2. 從 main 建立 feature branch
  3. 進行變更，並盡可能在 E-Ink 裝置上測試
  4. 送出 Pull Request 並清楚描述變更內容

本專案使用 Fastlane 進行部署自動化，並支援 F-Droid 發佈。
